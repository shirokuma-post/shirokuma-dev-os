# 成熟度ロードマップ — 何も削らず、順序を付ける

> Tier: T1 ・ 1行責務: guardrails の全項目（残ギャップ含む）を Now/Next/Later に段階化し、「不要だから省く」ではなく「順番に全部入れる」を保証する
> 思想: 全部いる。ただし一度に全部は「幹から/お手本1個で完了にしない」に違反する。だから**段階化**。各段は前段が緑になってから進む。判定軸 = **不可逆 ＞ 可逆**、**コスト低×価値高 から**。

## Now（立ち上げ〜初期。安く・不可逆を可逆化）
最初の番人。ここが緑になるまで次に行かない。
- 秘密スキャン（pre-commit ＋ CI）／`.gitleaks.toml`・`lefthook`
- CIゲート: typecheck / lint / **本番ビルド job** / any ラチェット
- 禁止パターン番人（越境・特権・型逃げ）`forbidden-patterns.sh`
- SCA: `dependabot.yml` ＋ `npm audit`（CVE検知・設定だけ）
- DB を持つなら: RLS無効テーブル検出・テナント越境SQL／**バックアップ有効化**
- **expand-contract 移行**の習慣化（破壊的1段DDL禁止）← 不可逆対策・最優先
- **即ロールバックの1コマンド化**＋戻す条件を runbook に（B2）
- 死活スモーク（`smoke.mjs`）＋ `/health` 契約
- 構造化ログ＋request_id（可観測性の土台）
- 秘密ローテ周期を数値で決める

## Next（有料顧客がつく〜ローンチ前後。安全を測れる形に）
- **復元演習(restore drill)** を定期実施（A1・テストしてないバックアップは無効）
- **feature flag**（ship dark ＋ kill switch・軽量実装でよい）
- SAST: **Semgrep ルール**（失敗カタログの AST 化）＋ CodeQL
- **DAST baseline**（ZAP を preview URL に）
- SLO 2〜3個＋エラーバジェット＋**バーンレート・アラート**（runbook付き）
- route別メトリクス（5xx率・p95）ダッシュボード
- maxDuration / 依存ドリフトの定期チェック
- ローンチ前チェックリスト（`launch-readiness.template.md`）を機械ゲート化
- **SBOM** 生成（syft 1コマンド・インシデント時に遡れる）
- 死活監視 cron を全コンポーネントへ（お手本1個で止めない）

## Later（規模・調達要件・チーム化が来たら）
- canary / blue-green デプロイ（一部トラフィック先行・基盤要）
- 分散トレース(OpenTelemetry)・飽和メトリクス・合成監視
- フル DAST（認証付き）・ファジング
- SLSA provenance・artifact 署名(cosign)（配布/調達要件が出たら）
- DB のカオステスト・フェイルオーバー演習
- KMS への鍵集約・自動ローテ・定期アクセスレビュー
- チーム化したら: コードオーナー・レビューSLA・正式なオンコール/インシデント指揮

## 進め方のルール
1. **段を飛ばさない**: Now が緑 → Next。不可逆対策(移行・復元・ロールバック)は Now に置いてある＝最初に固める。
2. **各項目に番人を1個**（失敗カタログの思想）。「気をつける」で済ませない。
3. **Later を"やらない"と読み替えない**: コスト×価値で後ろにいるだけ。トリガー（顧客数・調達要件・チーム化）が来たら前倒す。
4. 進捗はこのファイルにチェックを入れて可視化（doc-constitution: 鮮度＞完全性で更新）。
