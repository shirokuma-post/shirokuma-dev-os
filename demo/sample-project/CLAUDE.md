## §0. 北極星（WHY）

- **北極星**: EC の購入体験を「金額が絶対に間違って表示されない」状態に保つ。金額表示の信頼が壊れると購入体験そのものが壊れる。
- **幹（構造の核）**: 金額計算・金額表示のロジック。ここが崩れると全画面に波及する。
- **危険度の測り方**: 変更の危険度は「金額の計算/表示の境界を侵すか」で測る。

## §0.5. 【最重要】Intent Anchor（意図の先出し宣言）

> AI の attention は目の前の Read 結果に引っ張られる（= perception hijack）。
> **意思決定の前**に、目的と触ってよい範囲を宣言して anchor する。

### 発火タイミング（義務）

- 新タスクを受けた**第 1 ターン**（必須）
- Read 結果を見て「scope 外だけど直したい」と思った瞬間（自己捕捉）

### 出力フォーマット

```
【Intent Anchor】
- 目的 = （§0 北極星に 1 行で照合）
- 今回の scope = （触っていい file path / 関数を具体列挙・最大限狭く）
- scope 外発見時 = 触らない・記録のみ
```

### 規律

- scope は依頼を**最大限狭く解釈**（拡張は依頼者の明示要求時のみ）
- scope 外の発見は**触らない default** → この demo では `FINDINGS.md` に記録する

## §1. 不変ルール（抜粋）

| ルール | 詳細 |
|---|---|
| **金額は常に最小単位（cents）で保持し、表示の瞬間だけ変換** | 途中計算で小数ドルにしない |
| **金額表示は 2 桁固定** | `$12.30` のように必ず小数第 2 位まで |
